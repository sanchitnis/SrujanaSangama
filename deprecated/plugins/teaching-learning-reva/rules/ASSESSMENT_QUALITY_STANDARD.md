# REVA University AI System Instructions — Assessment Quality Standard

This document details the quality benchmarks and rules for engineering assessments, exams, assignments, and practical labs. The AI agent co-works with faculty to audit and flag assessment designs to ensure high-academic integrity and deep student engagement.

---

## 1. AI-Triviality Bypass Audit

In the generative AI era, assessments that ask for simple recall or direct code snippets can be solved in seconds using basic AI prompts. These are classified as **AI-Trivial Tasks** and are prohibited for off-site continuous assessment.

### AI-Triviality Indicators:
*   Direct definition requests (e.g., "What is a binary search tree?").
*   Standard algorithms without context (e.g., "Write a Python program to sort a list using quicksort.").
*   Simple code templates that are widely available in online tutorials.

### Standard for Off-site Tasks:
Every off-site assignment or project must be redesigned to be **AI-resistant** by:
1.  **Adding Constraints**: Enforcing highly specific, localized constraints (e.g., "Modify the quicksort algorithm to prioritize elements from a specific student roster array").
2.  **Requiring Meta-Cognition**: Requiring students to submit a written self-reflection or audio/video explanation detailing *why* they chose their approach, *what* failures they encountered, and *how* they addressed them.
3.  **Real-world Contextualization**: Linking questions to Indian industry case studies, local environmental datasets, or custom simulated stakeholder requirements.

---

## 2. Assessment HOTS Dominance ($\ge 60\%$)

Mid-semester exams (CIA) and final end-semester exam blueprints must be audited for cognitive level distribution.

*   At least **60%** of the total marks in any major exam must belong to **HOTS (Higher-Order Thinking Skills)** categories (Bloom's Levels 4, 5, or 6).
*   Lower-order questions (Levels 1 and 2) must be limited to a maximum of **20%** of total marks.
*   Every question in a question bank must be explicitly tagged with its target Course Outcome (CO) and Bloom's Cognitive Level.

---

## 3. Assignment Enjoyability & Motivation

We believe that learning is most effective when it is challenging yet highly engaging. The agent will audit faculty assignments using the **REVA Assignment Enjoyability Matrix**:

*   **Autonomy / Creative Latitude**: Does the assignment allow students to choose their own domain, programming language, or artistic style?
*   **Real-world Impact**: Is the assignment outcome a tool that can be used by others, a public repository, or a community service prototype?
*   **Micro-Achievements**: Are there clear, satisfying intermediate milestones rather than a single massive, intimidating deadline?

---

## 4. Balanced Assessment Framework

Faculty must implement a hybrid assessment structure that balances unproctored, AI-assisted work with rigorous in-person validation.

| Assessment Type | AI Allowed? | In-person Validation Requirement |
| :--- | :--- | :--- |
| **Course Projects / PBL** | Yes (as helper) | **Mandatory Viva-Voce (oral exam)** using customized Socratic questioning pools to verify individual coding/concept contribution. |
| **Weekly Practical Labs** | Yes (as editor) | **Live execution & Modification**: Students must modify a random part of their code on the spot under lab faculty proctoring. |
| **Mid-Sem / End-Sem Exams**| No (strictly proctored)| Handwritten or locked-terminal execution. |
