# /case-brief [case name or citation]

**Usage:** `/case-brief Kesavananda Bharati v. State of Kerala` or `/case-brief AIR 1973 SC 1461`

Produce a structured case brief scaffold in the student's preferred format. In **drill-me** mode, prompt the student to fill each section from their own reading and provide feedback. In **explain-to-me** mode, walk through each section and explain the reasoning — but still never write the full brief for the student.

---

## Profile Check

- Read `case-brief-format` from student profile: IRAC / FILAC / Indian Narrative / custom
- Read learning style: drill-me vs. explain-to-me
- Default format if no profile: Indian Narrative (most common for REVA Law exam answer-writing)

## Format Templates

```
CASE BRIEF — [CASE NAME]
Citation: [full citation] [model knowledge — verify on Indian Kanoon / SCC Online]
Court: [court name]
Year: [year]
Coram: [judges, if known]

FACTS
[Student fills this in]

ISSUES
[Student fills this in — number each issue]

ARGUMENTS — PETITIONER/APPELLANT
[Student fills this in]

ARGUMENTS — RESPONDENT
[Student fills this in]

JUDGMENT
[Student fills this in — majority / dissent if applicable]

RATIO DECIDENDI
[Student fills this in — the binding legal principle]

OBITER DICTA
[Student fills this in — if any notable obiter exists]

SIGNIFICANCE / IMPACT
[Student fills this in — why this case matters; later developments]
```

### IRAC Format
```
CASE BRIEF — [CASE NAME]
Citation: [citation] [model knowledge — verify]

ISSUE
[The legal question the court had to decide]

RULE
[The rule of law applied]

APPLICATION
[How the court applied the rule to the facts]

CONCLUSION
[The court's holding]
```

### FILAC Format
```
CASE BRIEF — [CASE NAME]
Citation: [citation] [model knowledge — verify]

FACTS
[Key facts only — not everything, just what mattered to the decision]

ISSUE
[The legal question]

[Applicable statute / precedent]

APPLICATION
[Court's reasoning]

CONCLUSION
[Holding + significance]
```

## Drill-Me Mode

1. Show the blank template in the student's preferred format.
2. Ask the student to fill in **one section at a time** — starting with Facts.
3. After each section the student submits, give feedback:
   - What they captured correctly
   - What they missed or got wrong
   - One follow-up question about that section before moving to the next
4. Never pre-fill a section. If the student asks "just fill it in for me," decline:
   > "Fill it in as best you can — even a rough attempt. I'll tell you what needs work. That's how you actually learn a case."
5. At the end, give an overall brief quality assessment:
   - **Strong** — all essential elements captured
   - **Workable** — core elements present, gaps noted
   - **Needs work** — key issues or ratio missed; specific improvements listed

---

## Explain-to-Me Mode

1. Walk through each section of the case briefly — facts, issues, arguments, judgment, ratio.
2. After each section, ask the student to summarise it back in their own words.
3. Ask at least one Socratic question about each section before moving on.
4. End with: "Now close your notes and brief this case from memory — just the ratio and the key facts."

---

## Citation Rule

All case details provided from model knowledge must carry:
```
[model knowledge — verify on Indian Kanoon / SCC Online]
```

If the student provides a case name that cannot be identified with confidence:
> "I'm not confident I can accurately recall this case. Please paste the relevant paragraph or the holding, and I'll help you brief it from there."

Never invent a citation. Never silently guess at a holding.
