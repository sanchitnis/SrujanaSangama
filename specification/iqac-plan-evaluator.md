\## PURPOSE OF THE SKILL



The purpose is NOT to improve an IQAC plan.

The purpose is to determine whether IQAC is:



\- Creating \*\*institutional inevitability\*\*

OR

\- Quietly becoming a \*\*compliance optimizer\*\*



This skill answers one core question:



> “If the Vice Chancellor were absent for 12 months,

would REVA still climb the rankings ladder?”



If the honest answer is \*no\*, the plan fails.



\---



\## CORE EVALUATION AXES (NON‑NEGOTIABLE)



\### AXIS 1: ROLE CLARITY — ARCHITECT vs OPERATOR

The skill checks whether IQAC:



✅ Designs systems, standards, ownership models  

❌ Runs processes, chases departments, compiles data  



\*\*Fail conditions\*\*

\- IQAC “implements” systems

\- IQAC “collects” data as a core task

\- Success depends on IQAC follow‑ups



> Verdict logic:  

If IQAC disappears and excellence collapses → IQAC is overreaching.



\---



\### AXIS 2: RANKING PHILOSOPHY — SHADOW vs TARGET

The skill evaluates whether the plan treats rankings as:



✅ Shadows of deep systems  

❌ Primary objectives to be optimized  



\*\*Red flags\*\*

\- Parameter‑wise NIRF targeting

\- Year‑over‑year score anxiety

\- Language like “improve score”, “maximize marks”



> World‑class institutions never chase shadows.



\---



\### AXIS 3: SYSTEM IRREVERSIBILITY

For every proposed initiative, the skill asks:



> “What system will prevent regression next year — without senior leadership intervention?”



A valid system must have:

1\. Role‑based ownership

2\. Automatic triggers

3\. Leading indicators

4\. Auditability without explanation



If ≥2 conditions are missing → the initiative is invalid.



\---



\### AXIS 4: OWNERSHIP DISTRIBUTION

The skill checks whether:



\- Research systems are owned by Research Office

\- Academic outcome systems by Schools

\- Industry \& consulting systems by the respective wing



IQAC must only:

\- Define the architecture

\- Enforce ownership

\- Audit system health



If IQAC owns outcomes → structural failure.



\---



\### AXIS 5: DATA → DECISION PIPELINE

Given the upcoming AI‑driven NAAC/NIRF ecosystem, the skill evaluates whether:



✅ Data is automated, real‑time, and decision‑linked  

❌ Data is episodic, manual, and submission‑oriented  



Any plan that assumes PDF‑based evidence is classified as \*\*obsolete on arrival\*\* \[6](https://education.economictimes.indiatimes.com/news/higher-education/revolutionary-naac-accreditation-system-to-launch-ai-driven-assessments-and-online-verification/122979470).



\---



\## RUTHLESS QUESTIONS THIS SKILL WILL ASK IQAC



1\. Which 5 institutional systems, if perfected, will move REVA from 151–200 to sub‑100 without mentioning NIRF?

2\. What currently works \*only because the VC is present\*?

3\. Which department’s under‑performance is being masked by IQAC effort?

4\. Where is IQAC confusing documentation quality with institutional quality?

5\. If we freeze hiring for two years, do these systems still function?



Refusal or hand‑waving answers are treated as failure signals.
