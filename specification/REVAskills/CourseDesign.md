# SKILL: Course Design (DESIGN Phase - ADDIE Model)

## 1. SKILL OVERVIEW
- **SKILL Name:** Course Design
- **ADDIE Phase:** Design (D)
- **Competency Level:** Advanced
- **Duration:** 8-10 weeks
- **Difficulty:** Intermediate to Advanced

## 2. SKILL DESCRIPTION
Comprehensive skill set for designing effective courses aligned with outcome-based education (OBE) principles, ABET accreditation standards, and institutional pedagogical frameworks.

## 3. LEARNING OUTCOMES FOR THIS SKILL
By completing this skill training, educators will:
- CLO1: Articulate clear, measurable course learning outcomes aligned with Bloom's taxonomy
- CLO2: Map course outcomes to program outcomes (CO-PO alignment)
- CLO3: Develop comprehensive course content structures and syllabi
- CLO4: Design assessment blueprints with multiple evaluation methods
- CLO5: Create performance rubrics and grading frameworks
- CLO6: Establish CLO achievement targets and success metrics

## 5. COMPONENT 1: COURSE LEARNING OUTCOMES (CLOs)

### 5.1 What are CLOs?
Course Learning Outcomes are specific, measurable statements describing what students should know, understand, and be able to do upon successful completion of the course.

### 5.2 Characteristics of Effective CLOs
- **SMART:** Specific, Measurable, Achievable, Relevant, Time-bound
- **Learner-centered:** Focus on what learners will do
- **Observable:** Can be assessed and measured
- **Aligned:** Connect to program outcomes and career requirements
- **Diverse:** Cover cognitive, affective, and psychomotor domains

### 5.3 Bloom's Taxonomy Alignment

| Level | Description | Action Verbs |
|-------|-------------|--------------|
| Remember | Recall facts and basic concepts | define, duplicate, list, memorize, recall, repeat, reproduce |
| Understand | Explain ideas or concepts | classify, describe, discuss, explain, identify, locate, recognize |
| Apply | Use information in a new situation | choose, demonstrate, dramatize, employ, illustrate, interpret, operate |
| Analyze | Draw connections among ideas | appraise, compare, contrast, criticize, differentiate, discriminate |
| Evaluate | Justify a stand or decision | argue, defend, judge, select, support, value, evaluate |
| Create | Produce new or original work | assemble, construct, create, design, develop, formulate, write |

### 5.4 CLO Writing Guide

**Template:** Students will be able to [ACTION VERB] [CONCEPT/SKILL] [CONTEXT, if applicable]

**Good CLO Examples:**
- Students will be able to **design** efficient **algorithms** for **real-world computational problems**
- Students will be able to **analyze** **database designs** using **normalization principles**
- Students will be able to **create** **interactive web applications** using **modern frameworks**

**Poor CLO Examples:**
- Students will understand algorithms (vague, not measurable)
- Students will know about databases (too broad)
- Students will learn programming (lacks specificity)

## 6. COMPONENT 2: CO-PO MAPPING

### 6.1 What is CO-PO Mapping?
CO-PO Mapping is the systematic alignment of Course Learning Outcomes (CLOs) with Program Learning Outcomes (POs) to ensure curriculum coherence and effective preparation of graduates.

### 6.2 Program Learning Outcomes (POs)

The 12 Standard Program Outcomes:
- **PO1:** Engineering Knowledge
- **PO2:** Problem Analysis
- **PO3:** Design/Development of Solutions
- **PO4:** Investigation
- **PO5:** Modern Tool Usage
- **PO6:** The Engineer and Society
- **PO7:** Environment and Sustainability
- **PO8:** Ethics
- **PO9:** Individual and Team Work
- **PO10:** Communication
- **PO11:** Project Management and Finance
- **PO12:** Lifelong Learning

### 6.3 Mapping Levels

| Level | Definition | Marking |
|-------|-----------|---------|
| **3: Mastery** | CLO directly & thoroughly addresses this PO | ✓✓ or 3 |
| **2: Developing** | CLO contributes to development of this PO | ✓ or 2 |
| **1: Introducing** | CLO introduces/touches upon this PO | → or 1 |
| **-: Not Addressed** | No connection between CLO and PO | - |

### 6.5 Steps to Create CO-PO Mapping

1. **List all CLOs** for your course
2. **Review each PO** definition carefully
3. **Analyze relationship** between each CLO and PO
4. **Assign mapping level** (3, 2, 1, or -)
5. **Document rationale** for each mapping
6. **Review for completeness** - ensure all POs are adequately addressed
7. **Validate with team** - peer review and feedback

## 8. COMPONENT 4: ASSESSMENT BLUEPRINT

### 8.1 What is an Assessment Blueprint?
An assessment blueprint is a comprehensive plan showing how course learning outcomes will be assessed using multiple methods, aligned with content units and CLO difficulty levels.

### 8.2 Assessment Principles
- **Alignment:** Assessments must match CLOs exactly
- **Validity:** Assessments must measure what they claim to measure
- **Reliability:** Assessments must produce consistent results
- **Fairness:** Assessments must be equitable for all learners
- **Comprehensiveness:** Multiple methods assess different competencies

### 8.3 Assessment Methods

| Method | Format | Best For | Advantages | Limitations |
|--------|--------|----------|-----------|------------|
| **Quiz** | Multiple choice, T/F | CLO1-CLO2 (Remember, Understand) | Quick feedback, objective scoring | Limited depth |
| **Assignment** | Problem sets, essays | CLO2-CLO3 (Apply, Analyze) | Application of concepts, creativity | Time-consuming to grade |
| **Test/Exam** | Multiple choice, short answer | CLO1-CLO4 (All cognitive levels) | Comprehensive, standardized | High anxiety, snapshot |
| **Case Study** | Real-world scenarios | CLO3-CLO4 (Analyze, Evaluate) | Authentic, complex thinking | Subjective evaluation |
| **Project** | Individual or group work | CLO4-CLO6 (Create, synthesis) | Integration, authentic learning | Resource-intensive |
| **Lab/Practical** | Hands-on activities | CLO4-CLO5 (Apply, Create) | Practical skills, experiential | Equipment needs |
| **Presentation** | Oral communication | CLO5-CLO6 (Synthesis, evaluation) | Communication skills, accountability | Anxiety factor |
| **Portfolio** | Collection of work | CLO1-CLO6 (All levels) | Growth tracking, evidence of learning | Time to compile |

## 9. COMPONENT 5: PERFORMANCE RUBRICS

### 9.1 What are Rubrics?
Rubrics are detailed scoring guides that describe the characteristics of work at different performance levels, providing clear expectations and consistent grading.

### 9.2 Types of Rubrics

**Holistic Rubric:** Single score for overall work quality
**Analytic Rubric:** Separate scores for different criteria

### 9.3 Rubric Development Steps
1. Identify criteria/dimensions to assess
2. Define performance levels (typically 4-5)
3. Write descriptors for each level
4. Ensure clarity and consistency
5. Test and refine

### 9.4 Generic Academic Rubric (4-Level)

| Criteria | Excellent (4) | Good (3) | Satisfactory (2) | Poor (1) |
|----------|--------------|----------|-----------------|---------|
| **Content Accuracy** | All concepts 100% accurate; demonstrates deep understanding | Most (85-95%) accurate; good understanding | Some (70-85%) accurate; gaps present | Many (< 70%) inaccurate; significant misconceptions |
| **Completeness** | All requirements met with exceptional additions | All core requirements thoroughly met | Most (80%+) requirements met | Many (<80%) requirements missing |
| **Critical Analysis** | Excellent synthesis; original insights; advanced thinking | Good analysis; connects ideas appropriately | Basic analysis; limited synthesis | Minimal or surface-level analysis |
| **Clarity & Organization** | Exceptionally clear, logical, professionally presented | Clear and well-organized | Somewhat clear; organizational issues | Unclear; poorly organized |
| **Application to Real-World** | Outstanding real-world relevance and applicability | Good connection to practical contexts | Attempted application; limited relevance | No apparent real-world connection |
| **Use of Evidence** | Extensive, relevant, properly cited evidence | Adequate relevant evidence with citations | Some evidence; citation inconsistencies | Little or no supporting evidence |

### 9.5 Discipline-Specific Rubric Examples

**Assignment/Problem-Solving Rubric:**

| Criteria | Excellent (4) | Good (3) | Satisfactory (2) | Poor (1) |
|----------|--------------|----------|-----------------|---------|
| **Problem Understanding** | Clear identification of all problem components | Identifies most problem components | Identifies some components; gaps | Misses major components |
| **Solution Approach** | Appropriate, efficient algorithm/method | Appropriate approach; minor inefficiencies | Partially appropriate approach | Inappropriate or flawed approach |
| **Mathematical/Technical Correctness** | All calculations/operations correct | Minor errors (< 5%) | Multiple errors (5-15%) | Major errors (> 15%) |
| **Explanation** | Clear, detailed explanation of reasoning | Explanation provided; mostly clear | Explanation present; unclear sections | Little/no explanation |

**Project Rubric:**

| Criteria | Excellent (4) | Good (3) | Satisfactory (2) | Poor (1) |
|----------|--------------|----------|-----------------|---------|
| **Scope & Complexity** | Highly ambitious; exceeds expectations | Appropriately scoped; challenging | Adequate scope; moderate challenge | Limited scope; minimal challenge |
| **Technical Quality** | Code/design/work is excellent; production-ready | Good quality; minor issues | Acceptable quality; some issues | Poor quality; significant issues |
| **Documentation** | Comprehensive, clear, professional docs | Adequate documentation; clear | Basic documentation; some gaps | Poor/missing documentation |
| **Testing/Validation** | Thorough testing; edge cases covered | Adequate testing; major cases covered | Minimal testing; some major gaps | Little/no testing |
| **Presentation** | Professional, compelling, well-delivered | Clear, well-organized, engaging | Acceptable; some clarity issues | Poorly presented; unclear |

## 10. COMPONENT 6: CLO ACHIEVEMENT TARGETS & SUCCESS METRICS

### 10.1 Setting Achievement Targets
Achievement targets establish the minimum percentage of students who should successfully meet each CLO at a specified performance level.

### 10.2 Determining Appropriate Targets

**Factors to Consider:**
- Course level (foundational vs. advanced)
- Student population characteristics
- Course difficulty and prerequisites
- Historical performance data
- Program/institutional benchmarks
- Accreditation standards (typically 70% minimum)

### 10.3 Benchmarking Guidelines

| Course Type | Typical Target | Rationale |
|------------|--------|------------|
| **Foundational** | 85-90% | All students should master basics |
| **Intermediate** | 75-85% | Most students should achieve core competencies |
| **Advanced** | 65-75% | Rigor increases; some challenge is expected |
| **Elective/Specialized** | 70-80% | Self-selected students; varied backgrounds |

### 10.4 CLO Achievement Targets Template

| CLO | Learning Outcome | Difficulty | Target (%) | Success Metric | Baseline Data |
|-----|-----------------|-----------|-----------|----------------|---------------|
| CLO1 | Fundamental concepts understanding | Low-Moderate | 85% | 85% of students score ≥75% on Quiz 1 | [Prior year data] |
| CLO2 | Concept application | Moderate | 80% | 80% of students score ≥75% on Assignments | [Prior year data] |
| CLO3 | Analysis & synthesis | Moderate-High | 75% | 75% of students score ≥70% on Case Studies | [Prior year data] |
| CLO4 | Complex problem solving | High | 70% | 70% of students score ≥70% on Project | [Prior year data] |
| CLO5 | Integration & creativity | High | 75% | 75% of students demonstrate competency in Lab | [Prior year data] |
| CLO6 | Capstone competency | High | 75% | 75% of students score ≥70% on Final Project | [Prior year data] |

### 10.5 Disaggregation & Analysis

Achievement targets can be disaggregated by:
- **Assessment method:** Different targets for different assessment types
- **Bloom's level:** Higher rigor = lower target initially
- **Student demographics:** Adjust for special populations if needed
- **Prerequisites met:** Lower performance for under-prepared students

## 11. PEDAGOGICAL STRATEGIES & INSTRUCTIONAL DESIGN

### 11.1 Teaching-Learning Methods Aligned with CLOs

**For CLO1-CLO2 (Remember, Understand):**
- Traditional lecture with structured notes
- Worked examples and demonstrations
- Discussion and clarification
- Practice problems

**For CLO2-CLO3 (Apply, Analyze):**
- Problem-based learning
- Case studies
- Lab experiments
- Group discussions

**For CLO4-CLO6 (Analyze, Evaluate, Create):**
- Project-based learning
- Capstone projects
- Real-world applications
- Authentic assessments
- Student research/inquiry

### 11.2 Bloom's Level-Appropriate Activities

| Bloom's Level | Teaching Methods | Assessment |
|--------------|-----------------|-----------|
| **Remember** | Lecture, Direct Instruction | Recall quiz, fill-in-blank |
| **Understand** | Discussion, Explanation, Analogy | Concept mapping, explanation |
| **Apply** | Guided practice, Tutorials, Labs | Problem-solving, application tasks |
| **Analyze** | Case studies, Debates, Discussions | Analysis essays, case study reports |
| **Evaluate** | Critiques, Discussions, Seminars | Position papers, evaluations |
| **Create** | Project-based, Research, Design | Portfolios, projects, presentations |

### 11.3 Active Learning Strategies
- Think-pair-share activities
- Minute papers
- Peer teaching
- Role-playing scenarios
- Simulations & games
- Inquiry-based projects

## 12. COURSE DESIGN QUALITY CHECKLIST

### 12.1 CLO Quality
- [ ] CLOs are specific and measurable
- [ ] CLOs use action verbs from Bloom's taxonomy
- [ ] CLOs are learner-focused (what students will do)
- [ ] CLOs are appropriate for course level
- [ ] CLOs are realistic and achievable
- [ ] At least 4-6 CLOs defined for course
- [ ] CLOs cover multiple Bloom's levels
- [ ] CLOs clearly communicated to students

### 12.2 CO-PO Mapping Quality
- [ ] All CLOs mapped to at least one PO
- [ ] Most POs addressed by at least one CLO
- [ ] Mapping levels are appropriate (3, 2, 1, -)
- [ ] Rationale documented for each mapping
- [ ] Team reviewed and validated mapping
- [ ] No CLOs are orphaned (unmapped)
- [ ] Balance across all 12 POs

### 12.3 Content Structure Quality
- [ ] Content logically organized and sequenced
- [ ] Prerequisites clearly identified
- [ ] All CLOs addressed in content
- [ ] Balance across units
- [ ] Real-world applications included
- [ ] Manageable workload
- [ ] Updated and current content
- [ ] Diversity & inclusion represented

### 12.4 Assessment Quality
- [ ] Multiple assessment methods used (≥3 types)
- [ ] Each CLO assessed at least once
- [ ] Assessment methods match CLO levels
- [ ] Clear assessment schedule
- [ ] Formative and summative balance
- [ ] Assessments are valid and reliable
- [ ] Rubrics clear and specific
- [ ] Grading policies transparent

### 12.5 Rubric Quality
- [ ] Clear criteria defined
- [ ] 4-5 performance levels
- [ ] Descriptive (not comparative) language
- [ ] Consistent terminology
- [ ] Can be applied consistently
- [ ] Aligned with CLOs
- [ ] Communicated to students
- [ ] Available for practice assessment

## 13. COMMON PITFALLS & HOW TO AVOID THEM

| Pitfall | Problem | Solution |
|---------|---------|----------|
| Vague CLOs | "Students will learn X" | Use specific action verbs; include observable behaviors |
| Over-ambitious CLOs | Too many (>8) or too complex CLOs | Limit to 4-6 CLOs; focus on essential competencies |
| Unmapped CLOs | CLOs don't connect to POs | Map every CLO to at least one PO with rationale |
| Misaligned assessment | Assessments don't match CLOs | Create assessment blueprint; align each method to CLOs |
| Poor rubrics | Rubrics are vague or inconsistent | Use descriptive language; pilot-test with faculty |
| Unrealistic targets | Targets too high or too low | Analyze historical data; benchmark against similar courses |
| Content overload | Too much content for available time | Prioritize; identify essential vs. nice-to-know topics |
| No connection to POs | Students don't see relevance | Explicitly connect assignments to real-world applications |
