Setup & User Guide  ·  v2.0

 

 

What OpenClaw Is

A prompt-native system that turns Claude into a deeply personalised agentic assistant. It remembers who you are, routes your requests to specialist agents, learns from every session, and grows smarter over time — all without a single LLM API call in code.

 

📄 Markdown

Everything Claude does is a file you paste

🐍 Scripts

Handle only mechanical tasks — zero LLM calls

🧠 Claude

Is the AI — you bring the conversation

2. Installation & First-Time Setup

 

2.1  Unzip the Package

Download openclaw-system-v2.zip and unzip it to a location you'll use regularly.

 

# macOS / Linux

unzip openclaw-system-v2.zip -d ~/Documents/

cd ~/Documents/openclaw-system

 

# Windows (PowerShell)

Expand-Archive openclaw-system-v2.zip -DestinationPath C:\Users\YourName\Documents\

cd C:\Users\YourName\Documents\openclaw-system

 

2.2  Install Python Dependency

OpenClaw scripts use only one third-party library — PyYAML — for reading the config file.

pip install pyyaml

 

ℹ️ All other scripts use only Python standard library modules (re, datetime, pathlib, sys). No other packages are needed.

 

2.3  Run the Init Interview

The init script asks you ~15 questions and creates all your memory files. It takes about 3 minutes.

 

python3 scripts/local/init.py

 

The script will ask for:

•     Your name, role, and organisation

•     Your location, timezone, and primary language

•     Core values and long-term goals

•     Areas of expertise and current projects

•     Preferred tone, response length, and format

•     Trust level for autonomous actions

 

When it finishes, these files will be populated with your answers:

File / Command

Purpose

memory/soul.md

Your stable identity — name, role, values, goals, expertise, style

memory/semantic/work.md

Current role, organisation, projects, tools

memory/procedural/writing-style.md

Tone, format, and document preferences

memory/procedural/code-style.md

Programming languages, conventions, anti-patterns

context/tasks.md

Empty task list, ready for entries

context/active-project.md

Project context template

memory/episodic/recent.md

Seeded with your init session entry

memory/habits/habits.md

Empty habit tracker, ready for entries

 

2.4  Verify the System

Run the health check to confirm all required files exist and no scripts contain API calls:

python3 scripts/local/health.py

 

A healthy system outputs all green checks. If you see warnings, they are usually about memory files not yet fully populated — normal after a fresh init. Errors indicate missing files.

 

2.5  Edit Your Memory Files (Optional but Recommended)

The init script creates good starting files, but you can enrich them immediately by editing them in any text editor.

 

File / Command

Purpose

memory/soul.md

Add more detail to your expertise map, values, and goals

memory/semantic/vocabulary.md

Add your domain-specific terms and acronyms

memory/semantic/relationships.md

💡 All memory files are plain Markdown. You can read, edit, and version-control them directly. There is no database or hidden state — what you see in the files is what OpenClaw knows.

 

3. The Session Loop

 

Every OpenClaw session follows the same four-step loop. Once you've done it twice it becomes second nature.

 

1

Build Your Context Block

Run context_builder.py — it reads all your memory files and outputs a formatted block ready to paste into Claude.

 

2

Open Claude and Paste the Session Opener

Paste ORCHESTRATOR.md followed by the context block output. Claude activates as OpenClaw and knows who you are.

 

3

Work — Paste Agent Prompts as Needed

Chat normally. When Claude routes to a specialist, paste that agent's prompt file. Switch between agents freely.

 

4

Close the Session and Save Memory

Paste SESSION_CLOSER.md — Claude outputs a SESSION_SUMMARY block. Run update_memory.py to save everything.

 

3.1  Step 1 — Build Context Block

From your openclaw-system/ directory:

python3 scripts/local/context_builder.py

 

The output is a formatted block that starts with === OPENCLAW CONTEXT BLOCK === and contains:

•     Your soul profile (name, role, expertise, preferences)

•     Open tasks — overdue items are highlighted at the top

•     Recent episodic memory (last 8 session entries)

•     Active habits and their current streaks

•     Your active project context

•     Writing style and code style preferences

•     The agent registry (so Claude knows what agents are available)

 

Copy the entire output to your clipboard. On most systems:

# macOS — pipe directly to clipboard

python3 scripts/local/context_builder.py | pbcopy

 

# Linux (requires xclip)

python3 scripts/local/context_builder.py | xclip -selection clipboard

 

# Or use the built-in flag (requires pip install pyperclip)

python3 scripts/local/context_builder.py --copy

 

ℹ️ The context block is typically 3,000–8,000 tokens depending on how much is in your memory. This fits comfortably within Claude's context window even on the free plan.

 

3.2  Step 2 — Paste the Session Opener

In Claude's chat box, paste in this order:

 

①

Paste the full contents of prompts/session/ORCHESTRATOR.md

②

On a new line, paste the === CONTEXT BLOCK === output from Step 1

③

Send the message to Claude

 

Claude will respond with a personalised greeting that includes your name, open tasks, and any habits due for check-in. You are now in a live OpenClaw session.

 

3.3  Step 3 — Work and Route to Agents

Chat with Claude normally. When you ask for something that belongs to a specialist, Claude will tell you which agent prompt to paste.

 

You say something like…

Claude says → paste this

"Help me draft an email to the team"

→ Writing Partner. Paste: prompts/agents/WRITING_PARTNER.md

"Research agentic AI frameworks"

→ Research Analyst. Paste: prompts/agents/RESEARCH_ANALYST.md

"Add a task: prepare report by Friday"

→ Task Manager. Paste: prompts/agents/TASK_MANAGER.md

"Help me write a Python data pipeline"

→ Code Architect. Paste: prompts/agents/CODE_ARCHITECT.md

"Teach me about RAG from scratch"

→ Learning Coach. Paste: prompts/agents/LEARNING_COACH.md

"I want to do my weekly reflection"

→ Reflection Facilitator. Paste: prompts/agents/REFLECTION_FACILITATOR.md

"I just finished my morning run"

→ Habit Tracker. Paste: prompts/agents/HABIT_TRACKER.md

"Brainstorm ideas for our new programme"

→ Idea Incubator. Paste: prompts/agents/IDEA_INCUBATOR.md

"Analyse this CSV data"

→ Data Interpreter. Paste: prompts/agents/DATA_INTERPRETER.md

"Draft our AI policy for Academic Council"

→ Academic Advisor. Paste: prompts/agents/ACADEMIC_ADVISOR.md

 

💡 You can skip Claude's routing and paste any agent prompt directly whenever you know which specialist you need. The context from the session opener carries through the whole conversation.

 

You can switch agents multiple times in one session. After pasting a new agent prompt, Claude switches to that specialist's behaviour while retaining all prior context from the conversation.

 

3.4  Step 4 — Close the Session

At the end of your session, paste the session closer to generate a machine-readable summary:

 

Paste the full contents of prompts/session/SESSION_CLOSER.md into Claude.

 

Claude will output a structured block that starts with ===SESSION_SUMMARY_START===. Copy the entire block including those delimiters.

 

Then run:

python3 scripts/local/update_memory.py

 

Paste Claude's SESSION_SUMMARY block when prompted, then type END on a new line. The script will:

•     Add the session narrative to memory/episodic/recent.md

•     Write new facts to the appropriate semantic or procedural memory files

•     Add captured tasks to context/tasks.md

•     Mark completed tasks as done

•     Update habit streaks in memory/habits/habits.md

•     Update context/current-session.md with open threads for next session

•     Log everything to logs/memory-changes.log

 

⚠️ If you forget to run update_memory.py after a session, you can still do it the next day — just paste the SESSION_SUMMARY you copied earlier. The script writes timestamps based on the DATE field in the summary, not today's date.

 

4. Daily Utility Scripts

 

These scripts handle mechanical tasks between sessions. None make LLM calls.

 

4.1  Start a New Session

python3 scripts/local/new_session.py

 

Run this at the beginning of each day or session. It:

•     Shows overdue and due-today tasks

•     Shows any threads carried over from last session

•     Resets context/current-session.md and context/scratchpad.md

•     Logs the session start to logs/sessions.log

 

4.2  Check Tasks

python3 scripts/local/task_check.py

 

Prints a colour-coded task view: overdue (red), due today (yellow), due this week (green). Useful for a quick morning briefing without opening Claude.

 

4.3  Log Habits from Terminal

python3 scripts/local/habit_checkin.py

 

Shows your active habits and their current streaks. Type the number(s) of habits completed today. Also supports logging misses with miss N.

 

Example interaction:

  1. Morning reading  (12d streak · last: 2026-05-07)

  2. Exercise         (5d streak · last: 2026-05-07)

  3. Evening journaling  (0d streak · last: never)

 

  → 1, 3

  ✅ Morning reading — 13-day streak

  ✅ Evening journaling — 1-day streak

 

4.4  Maintain Episodic Memory

python3 scripts/local/prune_episodic.py

 

Run when memory/episodic/recent.md gets large (over 500 lines). Automatically archives the oldest entries to a monthly file (e.g. memory/episodic/2026-05.md) and keeps recent.md lean.

 

💡 The health check will warn you when recent.md is approaching the prune threshold, so you always know when to run this.

 

4.5  Health Check

python3 scripts/local/health.py

 

Validates the entire system: all required files exist, soul.md is populated, no scripts contain API calls, and logs are in place. Run this after major changes or if something feels off.

6.2  How Memory Gets Updated

Memory updates happen in two ways:

 

Automatic — via update_memory.py

After each session, when you run update_memory.py with Claude's SESSION_SUMMARY, the script:

•     Parses structured fields from the summary block

•     Writes new facts to the file specified in each MEMORY_UPDATE line

•     Adds tasks to tasks.md in the correct priority section

•     Updates habit streaks in habits.md

•     Appends the session narrative to episodic/recent.md

 

Manual — direct file editing

You can edit any memory file at any time in a text editor. This is encouraged for:

•     Bulk-adding relationships or vocabulary entries

•     Correcting inaccurate facts Claude learned

•     Adding historical context Claude wasn't present for

•     Removing outdated information (use [DEPRECATED date] rather than deleting)

 

Via Memory Steward — mid-session

During a session, paste MEMORY_STEWARD.md when you want to:

•     Explicitly tell Claude to remember something: "Remember that my new head of department is Dr. Sharma"

•     Ask what Claude knows about a topic: "What do you have on my accreditation timeline?"

•     Correct a fact: "Update — I no longer use Notion, switched to Obsidian"

Claude will emit [MEMORY: ...] markers that get saved when you run update_memory.py at session end.

 

6.3  The Marker Protocol

Claude appends structured markers at the end of responses. These are parsed by update_memory.py.

 

Marker

What it does

[MEMORY: fact | tier | file]

New fact learned — will be written to the specified memory file

[TASK: p1|p2|p3 | text | due]

Task captured — will be added to context/tasks.md

[HABIT: name | done/missed]

Habit check-in — will update streak in habits.md

[DEPRECATED: old fact]

Old fact marked invalid — will be soft-deleted

[SKILL_GAP: name | desc]

New agent needed — paste SKILL_GENERATOR.md to create it

 

ℹ️ Markers only appear at the end of Claude's response and only when applicable. They are not shown to you by default — they get processed by the script. If you want to see them, look at the raw response before running the script.

 

7. Extending OpenClaw

 

7.1  Creating a New Agent

When no existing agent handles what you need, paste the Skill Generator workflow to have Claude create one:

 

1.   Open a Claude session with the Orchestrator and context block loaded

2.   Paste prompts/workflows/SKILL_GENERATOR.md

3.   Fill in the REQUEST section in that prompt (agent name, what it does, trigger phrases)

4.   Claude outputs two files: a SKILL.md definition and a prompts/agents/ prompt file

5.   Copy each file to its correct location in the openclaw-system/ folder

6.   Add one row to agents/registry.md with the agent's name, file path, and trigger description

7.   The new agent is immediately usable — paste its prompt file in any session

 

💡 Generated agents follow the same structure as built-in agents and integrate with the same memory and marker system. The Skill Generator can also extend existing agents — describe what you want added and it will output a revised version.

 

7.2  Editing Existing Agents

All agent prompts are plain Markdown in prompts/agents/. Open any file in a text editor and modify it directly.

 

Common customisations:

•     Adding a domain-specific output template to a Writing Partner

•     Adding institutional-specific regulations to the Academic Advisor

•     Adjusting the Task Manager's priority thresholds

•     Adding a new creativity framework to the Idea Incubator

 

7.3  Version Controlling Your System

Because everything is plain files, OpenClaw works naturally with Git:

cd openclaw-system

git init

git add .

git commit -m "Initial OpenClaw setup"

 

Commit after each session's memory update to maintain a complete audit trail of what the system has learned. You can diff memory files to see exactly what changed.

 

💡 Add the logs/ directory to .gitignore if you don't want verbose session logs in your commit history. Everything important is in the memory/ and context/ directories.

 

8. Tips, Patterns & Troubleshooting

 

8.1  Getting the Most Out of Sessions

•     Be explicit about routing early. If you know you need the Code Architect, paste it immediately after the session opener rather than waiting for Claude to suggest it.

•     Keep sessions focused. OpenClaw works best when a session has a clear primary intent. Multi-topic sessions still work, but focused sessions produce cleaner memory updates.

•     Use workflow prompts for weekly rhythm. WEEKLY_REVIEW.md is designed to replace your Orchestrator session once a week. It covers reflection, tasks, habits, and priorities in one structured flow.

•     Tell Claude things explicitly. Say "remember that..." or "I prefer..." — Memory Steward will catch it and emit a marker. Don't assume Claude inferred a preference.

•     Paste data directly. For Data Interpreter sessions, paste your CSV or JSON directly into the conversation after the agent prompt. Claude will analyse it inline.

 

8.2  Context Window Tips

•     The context block is sized to fit comfortably in Claude's context window

•     If Claude says the context is too long: edit context_builder.py to reduce max_lines for less-used sections (e.g. vocabulary, ideas index)

•     For very long sessions with many agent switches, paste a short context refresh partway through: just the soul profile and active tasks sections

•     The context block shows an estimated token count at the top — useful for monitoring

Memory file is outdated

Edit it directly in a text editor — all files are plain Markdown, always editable

Context block is too long

Edit context_builder.py — reduce max_lines for sections like vocabulary or ideas index

Agent ignores my style prefs

Run python3 scripts/local/init.py to regenerate missing memory templates

Make sure you copy the full block including ===SESSION_SUMMARY_START=== and ===SESSION_SUMMARY_END===

Habits not updating

Check the format: habit_checkin.py expects active habits under '## Active Habits' in habits.md

 

8.4  Recommended Weekly Rhythm

 

Daily (morning)

python3 scripts/local/new_session.py — see overdue tasks and carried threads

Each session

context_builder.py → paste ORCHESTRATOR.md → work → SESSION_CLOSER.md → update_memory.py

Daily (evening, optional)

python3 scripts/local/habit_checkin.py — log habit completions for the day

Weekly (Friday)

Use WEEKLY_REVIEW.md workflow instead of the standard session opener

Monthly

Review soul.md — update goals and expertise map if anything has shifted significantly

When episodic memory is large

python3 scripts/local/prune_episodic.py — archive old entries

When you need a new agent

Use SKILL_GENERATOR.md workflow — takes ~5 minutes, agent is ready immediately
