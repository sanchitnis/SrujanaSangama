# /activity-design-ai-ready
## Design AI-Ready, AI-Proof Learning Activities

> Paste this command: `/activity-design-ai-ready <course-code> <unit-number> <target-cos>`
>
> **Purpose**: Design 3–5 unit-level learning activities that are AI-resistant, HOTS-aligned, authentic, and produce portfolio evidence. Activities are designed assuming students have AI tools; assessment focuses on judgment, contextual adaptation, ethical reasoning, and meta-cognition.
>
> **Session Type**: Activity design workshop | **Duration**: 60–90 minutes
>
> **Human Role**: You specify the unit context and COs. AI generates activity drafts with AI-readiness indicators. Faculty adapts to local context and student population.

---

## Input Requirements

Provide:

1. **Course Code & Unit Info**
   - Course code, title, semester
   - Unit number, unit topic/title
   - Estimated contact hours (18, 24, 27 hours typical)

2. **Target Outcomes & Bloom's Levels**
   - Which COs does this unit address?
   - Target Bloom's levels (L2, L3, L4, L5, L6)?
   - Any enterprise skills focus (initiative, collaboration, ethics, communication, creativity)?

3. **Prerequisites & Student Context**
   - Prior courses/concepts students should know
   - Student year (1st, 2nd, 3rd, 4th)
   - Estimated class size, lab availability, access to tools

4. **Assessment Strategy**
   - What proportion of course grade does this unit represent?
   - Is a portfolio output expected from this unit?
   - Any proctoring constraints (in-lab only, online, hybrid)?

## Activity Design Process

### Phase 1: Brainstorm Activity Options (AI Generates Drafts)

AI generates **3–5 activity options** across different pedagogical strategies:

#### Activity Type 1: In-Class Active Learning (Small Group or Whole Class)
**Strategy**: Flipped classroom, think-pair-share, jigsaw, Socratic seminar, case study discussion

**Template**:
```
Activity 1.1: [Activity Name]
────────────────────────────
Duration: [#] minutes
Format: [Flipped / Think-Pair-Share / Jigsaw / Socratic / Case Study]
Target COs: [CO#, CO#]
Bloom's Level: [L#]
Enterprise Skills: [e.g., Collaboration, Communication]

Setup:
- Pre-class preparation: [What students do before class]
- In-class process: [Step-by-step facilitation]
- Debrief & synthesis: [How facilitator brings insights together]

Materials Needed: [List]
Facilitation Tips: [How to handle common confusions, keep pace, ensure participation]

Assessment Indicator: 
- Peer feedback form
- Facilitator observation checklist
- Exit reflection (3-2-1: 3 ideas, 2 questions, 1 aha moment)

Portfolio Link: Does this activity produce a portfolio artefact?
- No (formative only)
- Yes: [Description of output]
```

#### Activity Type 2: Individual or Small Group Project/Problem
**Strategy**: Guided project, design challenge, case analysis, computational lab

**Template**:
```
Activity 2.1: [Project/Problem Name]
──────────────────────────────────
Duration: [#] hours (in-class + outside)
Format: [Guided / Open-Ended / Challenge-Based]
Target COs: [CO#, CO#]
Bloom's Level: [L#]
Enterprise Skills: [e.g., Initiative, Adaptability, Creativity]

Problem/Challenge Brief:
[Clear, contextual problem statement with real-world or domain-specific framing]

Constraints (AI-Readiness Indicators):
- Localization constraint: [Student-specific or context-specific element]
  Example: "Use your own surname or local business as dataset"
- Meta-cognitive requirement: [How must students reflect on their process?]
  Example: "Explain why you chose this algorithm over alternatives; document one failed attempt"
- Authenticity indicator: [How does this task reflect real professional/academic work?]
  Example: "Your solution will be reviewed by an industry mentor"

Scaffolding / Milestones:
- Milestone 1 (Week X): [Deliverable with checkpoint feedback]
- Milestone 2 (Week X): [Deliverable]
- Final Submission (Week X): [Full artefact]

Rubric (Analytic):
| Criterion | Excellent (4) | Good (3) | Satisfactory (2) | Poor (1) |
| Technical Solution | Correct, efficient, well-tested | Mostly correct, minor optimization issues | Several errors, basic functionality | Major errors, non-functional |
| Problem Analysis | Deep understanding of domain constraints | Adequate analysis | Basic analysis | Minimal analysis |
| Meta-cognition Reflection | Clear reasoning about choices; documented learning | Reasoning present; some reflection | Minimal reflection | No reflection |
| Creativity/Originality | Novel approach, unexpected insight | Some originality | Standard approach | No originality |

Portfolio Output:
- Project document / code / prototype
- Reflective narrative: "How did I approach this problem? What did I learn? What would I do differently?"
- Peer or instructor feedback (if collaborative)

AI-Triviality Audit Questions (Faculty to verify):
- [ ] Could a student submit this entirely AI-generated without modification and pass?
- [ ] Does the rubric have observable criteria that distinguish authentic understanding from generic AI output?
- [ ] Are there contextual constraints that force the student to adapt/personalize the solution?
- [ ] Is there a meta-cognitive or viva component that surfaces individual understanding?
```

#### Activity Type 3: Peer Teaching / Viva-Voce (Proctored)
**Strategy**: Student teaches peer; live Q&A; real-time modification challenge

**Template**:
```
Activity 3.1: [Peer Teaching or Viva Session]
─────────────────────────────────────────────
Duration: [15-30 min per student]
Format: [Peer Teaching / Oral Exam / Live Coding / Real-time Problem Solving]
Target COs: [CO# (usually Strength 3 CO-PO mappings)]
Bloom's Level: [L4-L6]
Enterprise Skills: [Communication, Ethical Reasoning]

Purpose:
Validate that the student achieved the CO themselves; not delegated to AI or peer.

Proctoring Scenario:
- Student is given [scenario / problem / question] with [#] minutes to prepare
- Student presents understanding or solution to peer or instructor
- Peer or instructor poses 2-3 follow-up questions (randomized from question bank)
- Student demonstrates real-time modification or problem-solving

Question Bank (Sample):
1. "Explain your reasoning in one sentence."
2. "What would you change if [new constraint]?"
3. "Show me where you would modify your code/solution if [edge case]."
4. "What was the most challenging part? How did you overcome it?"
5. "How does this relate to the concept we covered in [prior unit]?"

Observation Rubric:
| Indicator | Present? | Evidence |
| Student articulates clear understanding | Y / N | [Specific phrase/explanation] |
| Student can modify/adapt on the spot | Y / N | [What was changed?] |
| Student connects to prior learning | Y / N | [Link to prior CO or concept] |
| Student acknowledges uncertainty appropriately | Y / N | [Phrases like "I'm not sure, let me think..."] |

Result:
- Pass: Student demonstrates understanding; CO marked as attained
- Refer: Student shows confusion; offer remediation; re-assess
- Inconclusive: Student performance unclear; conduct second viva
```

#### Activity Type 4: Reflective / Metacognitive Component
**Strategy**: Journaling, self-assessment, peer feedback, ethical case reflection

**Template**:
```
Activity 4.1: [Reflection or Metacognition Task]
───────────────────────────────────────────────
Duration: [#] minutes + [#] for feedback
Format: [Written Reflection / Video Reflection / Peer Feedback Circle]
Target COs: [CO#, particularly Enterprise Skills COs]
Bloom's Level: [L5-L6]
Enterprise Skills: [Ethical Reasoning, Self-awareness, Communication]

Prompt:
[Open-ended question that requires meta-cognition, ethical reasoning, or value clarification]

Example Prompts:
- "Reflect on a decision you made in this project. What did you consider? Who did your decision affect? Would you decide differently now?"
- "What was the hardest part of this unit? How did you approach learning it? What would help you learn it better next time?"
- "Imagine this software/system you built is used by [vulnerable population]. What ethical considerations should the designer have thought about?"
- "Explain one concept from this unit to someone who has never studied it. What analogies or examples did you use? Why were they effective?"

Submission Format:
- Written (500–800 words)
- Video (3–5 min)
- Poster + peer presentation

Rubric (Holistic):
| Level | Description |
| Sophisticated | Reflects deeply on learning; acknowledges multiple perspectives; connects to broader context; shows growth |
| Proficient | Reflects clearly on learning; considers impact; shows self-awareness |
| Developing | Reflection present but surface-level; limited acknowledgment of complexity |
| Emerging | Minimal reflection; largely descriptive |

Portfolio Value: Yes — reflections often showcase student growth and employability-relevant meta-cognition
```

#### Activity Type 5: Portfolio Synthesis / Cumulative Task
**Strategy**: Integration project, capstone mini, portfolio piece assembly

**Template**:
```
Activity 5.1: [Portfolio Synthesis / Integration Task]
──────────────────────────────────────────────────────
Duration: [Full unit or 3-4 weeks in context of larger course]
Format: [Synthesis Project / Capstone / Portfolio Piece]
Target COs: [2-3 major COs from unit]
Bloom's Level: [L5-L6]
Enterprise Skills: [Creativity, Value Creation, Initiative]

Synthesis Challenge:
[Real-world or domain-integrated problem that requires combining multiple unit concepts]

Example:
"Design a system that addresses [local problem or global challenge] using [unit concepts]. Your solution must be technically sound, ethically justified, and feasible with current resources."

Deliverable:
- Technical document (design, architecture, implementation)
- Reflective narrative (how you synthesized concepts; what you learned)
- Proof of concept or prototype (where feasible)
- Presentation or demonstration (to peer or stakeholder if possible)

Rubric (Analytic - Major Criteria):
| Concept Integration | Technical Quality | Ethical Grounding | Communication | Innovation |
| All unit concepts applied appropriately | Solution is sound and implementable | Ethical implications identified and addressed | Clear presentation; audience can follow reasoning | Approach is creative or novel |

Portfolio Status: YES — This is typically a course-level portfolio artefact
Grading Weight: [#]% of course grade
```

### Phase 2: AI-Readiness Verification Checklist

For each activity designed, verify it is AI-resistant and authentic:

#### Checklist: Is This Activity AI-Ready?

**Constraint & Contextualization** ✓
- [ ] Activity includes a **localization constraint** (student-specific, roster-specific, or context-specific element) that makes AI output generic/useless without student adaptation
- [ ] Constraint is non-trivial; not just "use your name" but rather "use your own data set / your local community / your past project"
- [ ] Constraint requires student judgment to apply meaningfully

**Meta-Cognition Component** ✓
- [ ] Activity requires student to **explain reasoning**, document **approach choices**, or **reflect on learning**
- [ ] Rubric explicitly assesses meta-cognitive element (not just solution correctness)
- [ ] Student must demonstrate how they *thought*, not just what they *produced*

**Authenticity Indicator** ✓
- [ ] Activity mirrors **real professional or academic practice** in the domain
- [ ] Deliverable is a **tangible artefact** (code, design document, prototype, analysis, publication, etc.) not just a grade
- [ ] Audience or user perspective is clear ("This will be reviewed by...")

**Proctoring Strategy** ✓
- [ ] At least one major assessment point includes **live validation**: viva-voce, real-time modification, peer teaching, or supervised lab
- [ ] Viva or live assessment has a **structured question bank** to reveal understanding

**Rubric Clarity** ✓
- [ ] Rubric uses **observable, measurable criteria** (not subjective judgments like "excellent" without definition)
- [ ] Rubric criteria distinguish **authentic understanding** from **generic AI output**
- [ ] Rubric explicitly includes **enterprise skills** (collaboration, communication, ethical reasoning) where relevant

**Assessment Authenticity** ✓
- [ ] Feedback is provided by **instructor, peer, or stakeholder** — not auto-graded
- [ ] Rubric requires **human judgment** to apply (L5-L6 thinking required to grade)

## Using This Workflow

### Scenario 1: Designing a New Course's First Unit Activities
1. Specify: Course, Unit 1, target COs (e.g., CO1, CO2), Bloom's L3-L4
2. Run `/activity-design-ai-ready`
3. Receive 5 activity drafts with AI-readiness indicators
4. Faculty reviews and customizes for local context
5. Write final activity playbook and share with course team

### Scenario 2: Refreshing an Existing Course's Weak Unit
1. Identify unit with low CO attainment or weak engagement
2. Run `/activity-design-ai-ready` with revised Bloom's/enterprise skills focus
3. Receive new activity suggestions with emphasis on proctoring or meta-cognition
4. Replace or supplement existing activities
5. Track CO attainment improvement next semester

### Scenario 3: Adding Enterprise Skills to Technical Courses
1. Specify: Course, Unit, target enterprise skill (e.g., Collaboration)
2. Run `/activity-design-ai-ready` with enterprise skills flagged
3. Receive activities with explicit collaboration or communication components
4. Customize peer teaching, team projects, or reflection tasks
5. Implement and assess enterprise skill development
