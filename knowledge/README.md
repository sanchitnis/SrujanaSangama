# knowledge/ — Course Knowledge Artefacts

This directory contains all course knowledge artefacts generated by the **Course Buddy Builder** pipeline. Every subfolder corresponds to one course and contains an Obsidian wiki, a Jupyter workbook, and flashcards.

> **Generated content** — do not edit generated pages directly.
> To update: edit the course descriptor and run `build.py --refresh`.

---

## Folder Structure

```
knowledge/
  [CourseCode]-[ShortName]/          e.g.  B25CI0201-ACP-GenAI/
    wiki/
      index.md                       Course overview, unit links
      [concept-slug].md              One page per key concept (beginner / intermediate / advanced)
      glossary.md                    All key terms
      concept-map.md                 Mermaid dependency diagram
      .obsidian/app.json             Turnkey Obsidian vault config
    workbook.ipynb                   Jupyter practice notebook (one section per unit)
    flashcards.md                    Markdown flashcards (if NotebookLM enabled)
    audio-overview.mp3               Audio overview (if NotebookLM enabled)
    faculty-notes/
      README.md                      How to add faculty notes
      [topic].md                     Faculty annotations (direct commit, no PR needed)
    student-contributions/
      README.md                      How to submit contributions
      [concept-slug]-[reva-id].md    Student-submitted explanations/examples
    CONTRIBUTING.md                  Contribution guide for this course
  instances/
    [CourseCode]-[ShortName]/
      skill.md                       Course Buddy skill — copy to agents/course-buddies/instances/
```

---

## Naming Convention

| Token | Rule | Example |
|-------|------|---------|
| `CourseCode` | University course code, exact | `B25CI0201` |
| `ShortName` | From `short_name` in descriptor frontmatter | `ACP-GenAI` |
| `concept-slug` | Lowercase, hyphen-separated | `dynamic-memory-allocation` |
| `reva-id` | Student's REVA ID | `1RV23CS001` |

## How Content is Generated

```
Faculty fills course descriptor
        ↓
build.py (course-buddy-builder)
        ↓
┌─────────────┬──────────────┬─────────────┐
│ wiki/       │ workbook     │ skill.md    │
│ (Obsidian)  │ (Jupyter)    │ (Socratic   │
│             │              │  tutor)     │
└─────────────┴──────────────┴─────────────┘
```

**Full pipeline** (with NotebookLM AI enrichment):
```powershell
cd skills\course-buddy-builder\tools\course-buddy-builder
python build.py --course-file templates\[CourseCode]-descriptor.md --output-dir ..\..\..\..\knowledge\
```

**Offline / template-only**:
```powershell
python build.py --course-file templates\[CourseCode]-descriptor.md --output-dir ..\..\..\..\knowledge\ --skip-notebooklm
```

**Refresh existing course**:
```powershell
python build.py --course-file templates\[CourseCode]-descriptor.md --output-dir ..\..\..\..\knowledge\ --refresh --gap-topics "concept A, concept B"
```

## Contributing

### Faculty
- Add teaching notes and annotations to `[CourseCode]-[ShortName]/faculty-notes/` as separate `.md` files.
- Direct commit — no PR required. You are the course owner.
- To update a generated wiki page, edit it directly and commit.

### Students
- Add worked examples, alternative explanations, or corrections to `[CourseCode]-[ShortName]/student-contributions/`.
- Name your file `[concept-slug]-[your-reva-id].md`.
- A faculty reviewer will evaluate and promote accepted contributions to the main wiki.
- **An accepted contribution is a Srujana Stage 2–3 portfolio evidence item** (teaching a concept to peers).
